Challenge: Winter is Coming

Challenge Objective

The objective of this challenge is to explore Model Steering on an image generation model, specifically Stable Diffusion, using the Activation Patching technique. Participants will need to manipulate the model's behavior to inject or remove the specific concept: winter.

Context: Stable Diffusion

Stable Diffusion is a latent diffusion model capable of generating high-quality images from textual descriptions (text-to-image). Its complex architecture allows for fine control over the generation process. To understand how it works, let's break down its main components:

  • Text Encoder: This component is responsible for translating the input text prompt into a numerical representation (embedding) that the model can understand. It converts human language into a rich, condensed vector space that captures the semantic meaning of the prompt. This embedding guides the image generation process.
  • U-Net: The U-Net is the core of the diffusion process. It iteratively refines a noisy latent representation into a coherent image. It takes the noisy latent image and the text embedding (from the Text Encoder) as input. Through a series of convolutional and deconvolutional layers, it learns to denoise the image step by step, guided by the textual description. The U-Net's "U" shape allows it to capture both low-level details and high-level contextual information.
  • Variational Autoencoder (VAE): The VAE has two main functions:
    1. Encoder: It compresses the original image into a smaller, latent representation. This is crucial because the diffusion process (handled by the U-Net) operates in this lower-dimensional latent space, which is computationally more efficient than working directly with high-resolution images.
    2. Decoder: It takes the refined latent representation from the U-Net and reconstructs it back into a high-resolution image. Essentially, it translates the abstract latent code into the visual pixels we see.

Methodology: Activation Patching and Required Tasks

Activation Patching is a technique used to understand and modify the internal representations (activations) of a neural network. It involves directly manipulating the activations at specific layers within the model's computational graph. By comparing and altering these internal states, we can observe and control how the model processes and generates specific features or concepts. This method provides a powerful way to perform "surgery" on the model's internal knowledge.

The central idea is to compare the model's activations when the concept "winter" is explicitly present in the prompt versus when it is absent. Participants will need to create a specific number of prompt pairs, denoted by k (where k is an integer greater than or equal to 1). Each pair will consist of:

  • Prompt: A prompt where the concept "winter" is clearly included (e.g., "A landscape in summer").
  • Negative Prompt: A corresponding prompt where the concept is omitted, while keeping other elements as similar as possible (e.g., "A landscape in winter").

For each of these k pairs, you will need to:

  1. Execute the forward pass of the model with both the "Prompt" and "Negative Prompt".
  2. Record the activations in different intermediate layers of the text encoder.
  3. Calculate the "Refusal Vector": This vector is derived by taking the difference between the activations obtained from the "Prompt" and those from the "Negative Prompt", for the chosen text encoder layers. This vector effectively isolates the specific information or "signature" associated with the concept you wish to manipulate.
  4. Apply the Refusal Vector: The calculated Refusal Vector will then be used to modify the activations of a new, target prompt (e.g., "a cozy cabin"). By adding or subtracting this vector, you can inject the concept (e.g., make the cabin appear in a winter setting) or remove an unwanted concept from the generated image.

Generate 20 Modified Images and Upload for Evaluation: After calculating the refusal vector, generate 20 images using the provided target prompts. Upload these images for evaluation. The evaluation will consider the presence of the generated concept, the coherence with the target prompt, and the perceptual quality of the image.

Rules

  • The challenge is open to groups of 3 to 5 people.
  • DO NOT change the seed of the code.
  • You can use any number of prompt/negative prompt pairs.
  • The prompts CANNOT be taken from the test prompts.
  • The code will be delivered and examined. Cheaters will be disqualified.

Recommended Tools

  • Python: Main programming language.
  • PyTorch: Deep learning frameworks for interacting with Stable Diffusion.
  • Hugging Face Transformers / Diffusers: Libraries for loading and using pre-trained Stable Diffusion models and accessing their internal components.
  • NNsight: A Python library for visualizing and interpreting neural networks, which can be useful for inspecting activations.
